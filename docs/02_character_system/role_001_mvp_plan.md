# role_001 MVP 计划

第一阶段目标是验证 role_001 是否能成为“类游戏角色编辑器”体系下的样板角色。

## 最小验证目标

- 准备 20 到 40 张精选图。
- 做 6 张角色标准图。
- 测试 2 套发型。
- 测试 2 套服装。
- 测试 2 种妆容。
- 判断是否需要角色 LoRA。

## 6 张角色标准图

建议输出：

- 正脸头像。
- 三分之二侧脸头像。
- 上半身正面。
- 上半身侧向。
- 全身正面。
- 全身姿势图。

## 两套发型

先用参考图和 prompt 测试，不急于训练发型 LoRA。

- 日常发型。
- 变化发型，例如双马尾、扎发或盘发。

## 两套服装

优先从 `outfits` 候选集中选择：

- outfit_001：旗袍/新中式方向。
- outfit_002：Lolita/甜系方向。

## 两种妆容

先做轻量测试：

- 日常淡妆。
- 更明显的舞台/写真妆。

## 是否需要角色 LoRA 的判断

如果 FaceID 或 IPAdapter 能稳定保持身份，可以暂缓训练角色 LoRA。

如果出现以下问题，则进入角色 LoRA 阶段：

- 每张图脸都变。
- 发型或服装一换就不像同一个人。
- 全身图和上半身图身份不一致。
- 需要批量生产分镜，FaceID 调参成本过高。
