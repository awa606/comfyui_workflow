# 角色资产化生产线策略

## 1. 为什么不建议训练一个万能人物 LoRA

不建议把角色身份、服装、妆容、发型、姿势和画风全部塞进一个万能 LoRA。万能 LoRA 看起来省事，但会把本来应该独立控制的因素绑死，后续很难像游戏角色编辑器一样单独调节。

常见问题：

- 想换衣服时，脸也跟着变。
- 想换妆容时，发型或年龄感被带偏。
- 想换发型时，眼镜、脸型或服装一起漂移。
- 想让同一角色进入不同场景时，背景和构图被训练集里的旧图污染。
- 想修复问题时，很难判断失败来自身份、服装、妆容、发型还是主模型。

角色资产化生产线应该把“身份”和“可替换外观”分离：身份 LoRA 负责让林薇一直像林薇，服装、妆容、发型、姿势和场景则用独立参考或后续独立 LoRA 控制。

## 2. 角色身份 LoRA、服装 LoRA、妆容 LoRA、发型 LoRA 的区别

### 角色身份 LoRA

角色身份 LoRA 学习稳定的人物身份：脸型、五官比例、眼镜、年龄感、气质和基础辨识度。它的目标是让角色在不同服装、姿势、光线和场景中仍然像同一个人。

### 服装 LoRA

服装 LoRA 学习固定服装结构和配色，例如粉色开衫、职业装、制服、鞋包搭配等。它不应该学习角色脸，否则换装时会污染身份控制。

### 妆容 LoRA

妆容 LoRA 学习眼妆、唇色、腮红、肤色质感和整体妆感。它适合在角色身份稳定后使用，用于区分日常妆、正式妆、舞台妆或特定剧情妆容。

### 发型 LoRA

发型 LoRA 学习发长、刘海、卷直、扎发、发色和发量轮廓。它适合在角色需要多个固定发型版本时使用，例如日常披发、低马尾、盘发或短发造型。

## 3. 哪些图片适合训练身份 LoRA

适合身份 LoRA 的图片应优先满足：

- 脸部清晰，五官和脸型可辨认。
- 黑框眼镜完整可见，眼睛没有被严重反光遮挡。
- 长深棕色头发与目标设定一致。
- 年龄感、脸型、气质稳定。
- 有正脸、三分之二侧脸、轻微侧脸。
- 有头像、上半身、半身或少量全身图。
- 背景简单，不抢主体。
- 服装不要过于单一，避免模型把某套衣服误学成身份。
- 表情自然，避免大量夸张表情。

身份 LoRA 的训练图核心是“这是谁”，不是“她穿什么、在哪里、摆什么姿势”。

## 4. 哪些图片只适合作为服装/姿势/场景参考

以下图片不适合直接进入身份 LoRA，但适合放入对应参考目录：

- 脸太小、脸模糊或五官不可辨认的全身图。
- 背影、侧背、低头、遮脸图。
- 重点是衣服结构、鞋包、饰品或配色的图片。
- 姿势很有价值但脸部质量差的动作参考图。
- 场景构图、室内空间、街景、办公室等环境参考图。
- 特殊光影、特殊镜头、特殊剧情氛围图。
- 妆容或发型很有参考价值，但身份脸不稳定的图片。

这些图应该进入 `outfit_refs`、`pose_refs`、`scene_refs`、`makeup_refs`、`hairstyle_refs` 或 `shoes_accessories`，用于生成时参考，而不是混入身份训练集。

## 5. 视频抽帧后如何筛选

视频抽帧只产生候选素材，不能直接作为训练集。

推荐流程：

1. 将原视频放入 `datasets/linwei/raw_videos`。
2. 运行 `scripts/extract_keyframes.py`，默认每秒抽 1 帧到 `datasets/linwei/extracted_frames`。
3. 先删除运动模糊、压缩噪点、闭眼、遮挡、脸太小的帧。
4. 再按用途分类：身份脸、上半身、全身、服装、发型、妆容、鞋包配饰、姿势、场景。
5. 对连续相似帧只保留 1 到 2 张，避免数据集重复。
6. 把不确定或低质量但暂时不想删除的图放入 `rejected`。

筛选时优先保留“身份稳定、清晰、角度多样”的图片，而不是保留最多的图片。

## 6. 上半身图、下半身图、全身图分别怎么用

### 上半身图

上半身图适合用于身份 LoRA 和服装上装参考。它通常能同时提供脸、发型、眼镜、肩颈、上衣和气质，是角色身份训练中最有价值的素材类型之一。

### 下半身图

下半身图更适合用于服装、裤裙、鞋子、包和配饰参考。由于通常缺少清晰脸部，不建议作为身份 LoRA 主力素材。

### 全身图

全身图适合建立身体比例、整体穿搭、姿势和构图参考。如果脸部足够清晰，可以少量用于身份 LoRA；如果脸太小，则应归入全身、服装或姿势参考。

## 7. 第一阶段只训练角色身份 LoRA 的理由

第一阶段只训练角色身份 LoRA，是为了先锁定最关键、最难修的部分：林薇到底长什么样。

理由：

- 漫画连续生产最怕同一角色每格像不同人。
- 身份稳定后，服装、妆容、发型和场景才能被可靠测试。
- 如果先训练万能 LoRA，会把服装和妆容错误绑定到身份上。
- 当前 FaceID 只是临时生成方案，资产化生产需要更稳定的身份锚点。
- 身份 LoRA 成功后，可以继续用 IPAdapter、ControlNet、inpaint 和 prompt 做局部变化。

因此第一阶段目标应是：让林薇在不同镜头、姿势和基础服装下仍保持同一身份，而不是一次性解决所有造型变化。
