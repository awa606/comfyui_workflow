# role_001 LoRA 训练决策

当前决策：不训练 LoRA。

本文件只用于记录后续是否需要训练角色身份 LoRA 的判断依据。

## 当前不训练的原因

- role_001 素材尚未导入。
- 视频尚未抽帧。
- 素材尚未人工筛选。
- manifest.csv 尚未生成。
- FaceID / IPAdapter 测试批次尚未完成。
- 还没有足够证据证明必须训练角色 LoRA。

## 优先验证顺序

1. 整理 role_001 原始素材。
2. 抽帧并筛选清晰身份素材。
3. 生成 manifest.csv。
4. 做 FaceID / IPAdapter 标准图测试。
5. 如果 FaceID 不稳定，再进入角色身份 LoRA 准备。

## 训练角色身份 LoRA 的触发条件

满足以下情况时才考虑：

- FaceID / IPAdapter 在多姿势、多服装下无法稳定保持角色身份。
- 已筛出 20-40 张高质量身份素材。
- 素材中有正脸、45 度、侧脸、半身、全身等变化。
- 服装、妆容、场景不会强绑定为单一模式。
- 用户确认该素材可以合法用于训练。

## 不训练万能 LoRA

不把以下内容混成一个万能 LoRA：

- 脸
- 身材
- 服装
- 发型
- 妆容
- 姿势
- 场景
- 漫画风格

原因：

- 内容互相绑定，后续无法像游戏角色编辑器一样调参。
- 换衣服容易改脸，换发型容易改身份。
- 风格和人物绑定后，迁移到不同漫画风格会变困难。
- 数据要求变高，失败原因更难定位。

## 分层策略

第一优先级：

- 角色身份 LoRA：只解决“这个角色是谁”。

后续可选：

- 服装 LoRA：当某套服装需要反复稳定出现时再做。
- 发型 LoRA：当发型需要作为可切换模块时再做。
- 妆容 LoRA：当妆容需要稳定复现且 prompt 难以控制时再做。
- 风格 LoRA：当漫画风格已经确定并需要批量统一时再做。
