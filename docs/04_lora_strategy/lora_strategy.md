# LoRA 策略

## 三类 LoRA 的区别

### 角色 LoRA

角色 LoRA 学习的是某个固定角色的身份特征，例如脸型、五官比例、发型、眼镜、年龄感、气质和整体辨识度。它的目标是让同一个角色在不同场景、姿势、镜头距离和服装下仍然像同一个人。

适合用于：

- 固定主角或重要配角。
- FaceID 无法稳定保持身份时。
- 长篇漫画需要同一角色反复出现时。

### 服装 LoRA

服装 LoRA 学习的是某套固定服装或造型，例如职业套装、校服、制服、外套、配饰组合和颜色搭配。它不应该负责学习角色身份，否则换人穿同一套衣服时会混入角色脸。

适合用于：

- 某个角色有长期固定服装。
- 多个角色需要共享同一类制服。
- 服装细节复杂，prompt 难以稳定描述。

### 风格 LoRA

风格 LoRA 学习的是画风、线条、上色、光影、质感、漫画分镜视觉语言等。它不应该绑定某个角色身份或具体服装，否则会降低泛化能力。

适合用于：

- 需要统一整部漫画的视觉风格。
- 主模型风格不稳定或不符合目标漫画方向。
- 已经确认角色和服装路线稳定后，需要统一最终画面质感。

## 为什么不建议训练一个万能 LoRA

不建议把角色、服装和风格全部塞进一个 LoRA，因为它会把不同概念互相绑定，导致后续控制变差。

常见问题包括：

- 想换衣服时，角色脸也跟着变化。
- 想换画风时，角色身份被削弱。
- 想让其他角色使用同一风格时，画面混入主角特征。
- 训练集稍微不平衡，就会把某个角度、表情、服装或光线误学成角色本体。
- 后续排查问题困难，不知道失败来自身份、服装还是风格。

更稳妥的路线是分层训练、分层验证：先角色，再服装，最后风格。每个 LoRA 只负责一个明确目标。

## 先训练角色 LoRA 的理由

当前阶段最关键的问题是角色一致性，而不是最终服装或最终画风。角色 LoRA 优先级最高，原因如下：

- 长篇漫画最怕主角每张图都像不同人。
- 角色身份稳定后，服装和风格才有可靠测试基准。
- FaceID 如果只能提供“相似”，角色 LoRA 可以补足更稳定的身份记忆。
- 角色 LoRA 的训练集要求相对明确，可以先用正脸、侧脸、半身、多表情素材建立基础版本。
- 先做角色 LoRA 可以判断当前 checkpoint 是否适合作为长期生产底座。

角色 LoRA 的目标不是记住某几张照片，而是学习“林薇”这个角色的稳定视觉身份。

## 什么时候训练服装 LoRA

满足以下条件后再训练服装 LoRA：

- 角色 LoRA 已能稳定保持脸型、眼镜、发型和年龄感。
- 已经确定角色常穿服装，并且 prompt 无法稳定复现关键细节。
- 漫画中同一套服装会高频出现。
- 服装不会频繁修改设计稿。
- 训练素材中可以清楚区分“角色身份”和“服装特征”。

如果服装还在设计阶段，优先用 prompt、参考图、ControlNet 或 IPAdapter 测试，不急于训练。

## 什么时候训练风格 LoRA

满足以下条件后再训练风格 LoRA：

- 主模型路线已经基本确定。
- 角色 LoRA 验证通过。
- 服装或造型路线没有重大不确定性。
- 已经收集到足够统一、合法可用、目标明确的风格参考图。
- 当前 checkpoint 无法稳定输出目标漫画线条、上色和镜头语言。

风格 LoRA 应尽量使用不绑定单一角色的素材，让它学习画面语言，而不是学习某个人或某套衣服。
