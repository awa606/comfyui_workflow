## 为什么不盲目升级大模型

当前瓶颈不是生成能力，而是素材资产是否可检索、可去重、可聚类、可人工复核。

不盲目升级大模型的原因：

- 大模型显存、磁盘和依赖成本高，容易让本地环境进入排错状态。
- 多个模型同时接入会让失败原因混杂，难以判断是素材、阈值、模型还是 pipeline 问题。
- 当前还未完成稳定的 `master_asset_index.csv`，先做基础索引比追逐更大的模型更有价值。
- FaceID / LoRA / ComfyUI 漫画化都依赖干净素材，而不是单纯依赖更强生成模型。

## InsightFace 当前保留的原因

InsightFace 保留，但只作为身份层。

当前用途：

- 为后续虚拟角色身份聚类提供 embedding。

不让 InsightFace 负责：

- 服装、鞋、道具检测。
- 人物或服装分割。
- 深度估计。

- LoRA 训练决策本身。

## 模型类别区别

生成模型：

- 输入 prompt、图像或控制条件，输出新图或编辑图。
- 例：SDXL、Pony、FLUX、Qwen-Image、Z-Image。
- 主要用于生成，不适合作为当前素材索引的第一步。

识别模型：

- 输入图像，输出标签、检测框、关键点或身份 embedding。
- 例：InsightFace、DWPose、OpenPose、GroundingDINO。
- 适合二次筛选系统。

分割模型：

- 输入图像和提示，输出 mask。
- 例：SAM2。
- 用于人物、服装、鞋、道具区域分离。

3D 模型：

- 输入图像或多视图，输出 mesh、深度、法线或 3D 草模。
- 例：image-to-3D 类模型。
- 当前只做调研，不进入完整 3D 生产。

## 为什么当前不把 FLUX/Qwen/Z-Image 作为第一主线

- 它们偏生成或图像编辑，不负责素材去重、聚类和结构化索引。
- 当前需要的是“识别和筛选”，不是更多随机生成结果。
- 过早切换到强生成模型会掩盖素材质量问题。
- 本地显存、依赖和节点兼容性还需要先小规模验证。
- FaceID / LoRA / 漫画化前，必须先把素材库整理干净。
