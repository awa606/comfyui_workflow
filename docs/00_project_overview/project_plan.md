# 项目计划

当前项目进入角色资产化准备阶段。总目标是从临时 FaceID 出图，升级为可复现、可维护、可扩展的漫画角色生产线。

## 阶段 1：素材整理

目标：

- 建立 `datasets/linwei` 角色资产目录。
- 收集但不上传真实图片和视频。
- 从视频中抽帧，人工筛选可用素材。
- 按身份、上半身、全身、服装、发型、妆容、鞋包、姿势、场景和废弃素材分类。
- 生成或维护 `metadata/dataset_manifest.csv`，记录每张图的用途和质量。

完成标准：

- 原始素材、抽帧、精选身份图和参考图分区明确。
- 每张候选图都能判断是否适合身份 LoRA。
- 不把真实图片、视频、模型文件提交或上传。

## 阶段 2：FaceID/模型测试

目标：

- 继续使用 FaceID 作为临时生成方案。
- 对 SDXL/Pony checkpoint 做横向比较。
- 测试 IPAdapter、ControlNet、FaceID 权重和 prompt 参数。
- 用 contact sheet 对比不同模型和参数下的角色一致性。

完成标准：

- 选出 1 到 2 个候选主模型。
- 明确 FaceID 的有效权重区间。
- 明确哪些失败来自模型、参考图、FaceID 方案或参数。
- 建立 `model_tests` 中的测试命名习惯。

## 阶段 3：角色 LoRA

目标：

- 只训练角色身份 LoRA，不训练万能 LoRA。
- 使用精选身份脸、上半身和少量高质量全身图。
- caption 保持稳定触发词和身份特征。
- 验证林薇在不同镜头、姿势、服装和场景下是否仍像同一个人。

完成标准：

- 角色身份稳定性明显高于 FaceID 临时方案。
- LoRA 权重有可用区间。
- 不把固定服装、妆容、场景错误绑定到身份上。
- 形成可复用的训练数据和参数记录。

## 阶段 4：服装/妆容模块

目标：

- 在身份 LoRA 稳定后，再拆分服装、发型和妆容控制。
- 优先用 prompt、IPAdapter、ControlNet 和 inpaint 测试。
- 只有当某套服装、妆容或发型需要高频复现时，才考虑独立 LoRA。

完成标准：

- 身份 LoRA 不被服装/妆容模块污染。
- 常用服装、发型、妆容有清晰素材来源和控制方式。
- 明确哪些模块需要 LoRA，哪些只需要参考图或 prompt。

## 阶段 5：分镜生产线

目标：

- 将角色身份、服装、姿势、场景和局部修复组合成分镜生产流程。
- 建立每格漫画的输入规范：角色、服装、姿势、镜头、场景、情绪、台词。
- 使用 ControlNet 控制姿势和构图。
- 使用 inpaint 修复眼镜、手部、脸部和服装细节。
- 使用统一风格方案保证整章视觉一致。

完成标准：

- 可以稳定生成同一角色的多格分镜。
- 每张图都有可追踪参数和素材来源。
- 失败图可以归因并回到对应阶段修复。
- 生产线从“碰运气出图”升级为“可控资产化生产”。
