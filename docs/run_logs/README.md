# Command Run Logs

This directory records key command runs for the project.

## Rules

1. Every key command run must have a Markdown run log.
2. Run logs can be committed to GitHub.
3. Local generated artifacts must not be committed, including `metadata`, `outputs`, JSONL, CSV, images, videos, NPZ files, and model files.
4. Each log must include the sections listed in `command_log_template.md`.
5. Logs should record the actual command, purpose, preconditions, outputs, metrics, problems, handling, next step, and Git safety check.

## Required Log Sections

- Basic Info
- Command
- Purpose
- Preconditions
- Output Summary
- Generated / Modified Files
- Key Metrics
- Problems Found
- Solution / Handling
- Next Step
- Git Safety Check
