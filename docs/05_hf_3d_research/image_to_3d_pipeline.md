# role_001 图像转 3D 流程

当前阶段只定义流程，不下载模型、不生成正式 mesh。

## 1. role_001 图片输入

优先从 `characters/role_001` 中选择 3 张测试图：

- 正面全身图。
- 三分之二角度全身图。
- 上半身或脸部较清晰图。

输入前检查：

- 主体完整，身体没有严重裁切。
- 姿势尽量简单，避免大幅遮挡。
- 服装轮廓清楚。
- 背景不要过于复杂。
- 如果只测试体型或服装体块，可以使用全身图；如果测试身份保持，需要脸部足够清晰。

## 2. 输入 image-to-3D 模型

测试时记录：

- 模型名称和版本。
- 输入图路径。
- 是否使用在线 Space、API 或本地脚本。
- 输出格式，例如 `.obj`、`.glb`、`.gltf`、`.ply`。
- 是否生成材质贴图。
- 运行时间和显存需求。

不要一次下载多个大模型。每次只选一个候选模型验证。

## 3. 输出 mesh 进入 Blender

Blender 导入建议：

- `.glb` / `.gltf`：优先导入，通常能带材质。
- `.obj`：需要同时检查 `.mtl` 和贴图。
- `.ply`：适合几何检查，但材质流程可能需要额外处理。

导入后先另存 Blender 工程副本，不覆盖原始输出。

## 4. Blender 中需要检查什么

基础检查：

- mesh 是否完整，有没有大面积破洞。
- 身高、头身比、肩宽、腿长是否合理。
- 法线方向是否正确。
- 材质是否丢失或错位。
- 头发、裙摆、鞋、手指是否严重粘连。
- 是否适合从正面、侧面、三分之二角度渲染。
- 是否能作为姿势和体型参考，而不是直接当最终角色资产。

如果要进一步进入角色编辑器路线，还需要检查：

- 是否可重拓扑。
- 是否可绑定骨架。
- 是否可拆分头发、服装、鞋袜、身体。
- 是否能与后续 ComfyUI 风格化保持一致。

## 5. 导出渲染图给 ComfyUI

Blender 输出给 ComfyUI 的内容：

- 正面渲染图。
- 侧面渲染图。
- 三分之二角度渲染图。
- 深度图或法线图。
- 姿势参考图。
- 透明背景人物图。

ComfyUI 使用方式：

- 渲染图作为 IPAdapter 或参考图。
- 深度图进入 Depth ControlNet。
- 姿势图进入 OpenPose/DWPose ControlNet。
- 透明人物图用于 inpaint、背景合成或漫画化。

最终目标是：3D 负责结构和角度一致性，ComfyUI 负责漫画风格、角色身份、服装细节和局部修复。
