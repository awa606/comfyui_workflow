# Hugging Face 模型调研计划

当前阶段只做调研、目录规划和脚本框架，不下载大模型、不训练模型、不上传图片、视频或模型文件。

## Hugging Face 在本项目中的作用

Hugging Face 在本项目中主要作为模型发现、模型卡审查、许可信息确认和在线 Space 预验证入口。它不是最终生产环境本身，而是帮助我们快速判断某类模型是否值得进入本地 ComfyUI/Blender 流程。

建议用途：

- 查找 image-to-3D、depth、pose、face identity、segmentation、image editing 等模型类别。
- 阅读模型卡、license、输入输出格式和硬件要求。
- 优先用在线 Space 或小模型验证效果。
- 记录模型是否已有 ComfyUI 节点或可接入脚本。
- 将模型输出是否能进入 Blender 或 ComfyUI 作为评估重点。

## 为什么不能只靠一个万能模型

类似游戏角色编辑器的角色资产系统需要多个可独立调节的模块。一个万能模型很难同时稳定处理人脸身份、姿势、服装、发型、妆容、分割、深度、3D mesh 和漫画化风格。

如果只依赖一个模型，常见问题包括：

- 人脸一致性和服装变化互相污染。
- 姿态控制不稳定，难以批量生成分镜。
- 图像转 3D 的 mesh 不适合直接漫画化。
- 编辑模型能改图，但不能替代角色资产管理。
- 出错后无法判断问题来自身份、姿势、服装、深度、分割还是风格化。

更稳的路线是分模块验证，确认每个模型负责的边界。

## image-to-3D 模型的用途和限制

用途：

- 从 role_001 的正面或全身图生成初步 mesh。
- 为 Blender 提供体型、姿势或服装体块参考。
- 作为 3D 角色基底路线的可行性验证。

限制：

- 单张图生成的背面、侧面和遮挡区域常常是模型猜测。
- 服装褶皱、头发、鞋、手指等细节可能不干净。
- mesh 拓扑可能不适合动画绑定或精修。
- 生成结果不一定保留真实人脸身份。
- 需要 Blender 中人工检查比例、法线、材质、尺度和可渲染性。

## depth 模型的用途

Depth 模型用于从图像估计空间层次和身体体块，帮助后续姿势、构图、3D 辅助或 ControlNet 控制。

适用场景：

- 从 role_001 标准图提取深度图。
- 给 ComfyUI 提供身体比例和场景空间参考。
- 辅助判断全身图是否适合进入 3D 或姿势控制流程。

## pose 模型的用途

Pose 模型用于提取人体关键点、手部、脸部或骨架控制图，服务于分镜姿势复用。

适用场景：

- 从参考图提取站姿、坐姿、手势和舞蹈动作。
- 给 ControlNet OpenPose/DWPose 使用。
- 将角色身份和姿势分离，避免用 prompt 硬控动作。

## face identity 模型的用途

Face identity 模型用于评估和保持角色身份一致性。

适用场景：

- FaceID / InstantID / IPAdapter FaceID 路线测试。
- 对比不同模型输出是否仍像 role_001。
- 为角色 LoRA 是否必要提供依据。

注意：Face identity 模型只能辅助身份保持，不能替代高质量身份素材筛选。

## segmentation 模型的用途

Segmentation 模型用于分离人物、头发、服装、腿部、鞋袜、背景和配饰。

适用场景：

- 从 role_001 图片中提取服装、头发、鞋袜区域。
- 为 inpaint、换装、背景替换和资产拆分提供 mask。
- 将训练素材拆成身份、服装、发型、妆容、鞋袜等模块。

## image editing 模型的用途

Image editing 模型用于对已有图进行语义编辑和局部修正。

适用场景：

- 修改服装颜色、局部配饰、妆容或背景。
- 辅助生成角色标准图变体。
- 对 ComfyUI 输出图做二次编辑对照。

它适合做编辑器工具，不适合作为第一阶段的唯一主线。

## 适合先测试的模型

优先测试：

- image-to-3D：先选一个在线 Space 或轻量 demo，验证 mesh 是否能进 Blender。
- depth：先选一个可快速出深度图的模型。
- pose：先选 DWPose/OpenPose 类流程，验证姿势图能否进 ComfyUI。
- face identity：先测 IPAdapter FaceID 或 InstantID 类工作流。
- segmentation：先测 SAM 类分割，验证服装、头发、鞋袜 mask 是否可用。
- image editing：先用在线或轻量编辑模型做单图编辑对照。

## 暂时不作为主线的模型

暂不作为第一主线：

- 需要超大显存、复杂依赖或下载巨大权重的模型。
- 只能在线使用且无法确认授权或复现流程的模型。
- 输出格式无法进入 Blender 或 ComfyUI 的模型。
- 只擅长单次惊艳生成、但无法控制角色资产的模型。
- 许可不清晰或不适合当前用途的模型。
